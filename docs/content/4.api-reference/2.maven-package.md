The VoxelDash API is published as a Java library (`de.gnm:voxeldash-api`). If you want to
build your own modules or integrations, you can pull it in as a dependency and work
directly against the same route classes, models and helpers that the dashboard uses. You
can find all of them in the [API Explorer](/api-reference/explorer).

The library is hosted on the VoxelDash Maven repository. You have to add the repository
and the dependency to your build, then refresh your project.

## Add the Dependency

::tabs{variant="card"}
::div{label="Maven" icon="simple-icons:apachemaven"}

Add the repository and dependency to your `pom.xml`:

```xml
<repositories>
  <repository>
    <id>voxeldash-maven</id>
    <url>https://packages.buildkite.com/voxeldash/maven/maven2/</url>
    <releases>
      <enabled>true</enabled>
    </releases>
    <snapshots>
      <enabled>true</enabled>
    </snapshots>
  </repository>
</repositories>

<dependencies>
  <dependency>
    <groupId>de.gnm</groupId>
    <artifactId>voxeldash-api</artifactId>
    <version>{{API_VERSION}}</version>
  </dependency>
</dependencies>
```

::

::div{label="Gradle (Kotlin)" icon="simple-icons:gradle"}

Add to your `build.gradle.kts`:

```kotlin
repositories {
    maven("https://packages.buildkite.com/voxeldash/maven/maven2/")
}

dependencies {
    implementation("de.gnm:voxeldash-api:{{API_VERSION}}")
}
```

::

::div{label="Gradle (Groovy)" icon="simple-icons:gradle"}

Add to your `build.gradle`:

```groovy
repositories {
    maven { url 'https://packages.buildkite.com/voxeldash/maven/maven2/' }
}

dependencies {
    implementation 'de.gnm:voxeldash-api:{{API_VERSION}}'
}
```

::
::

## Versioning

The repository serves both releases and snapshots. If you want stable builds, pin a fixed
version like in the examples above. If you'd rather follow along with the latest changes,
you can use a snapshot instead. Either way, the [API Explorer](/api-reference/explorer) is
generated from this exact library, so it always matches the version you're working with.
